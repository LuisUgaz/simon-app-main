# Flujo de Trabajo Conductor

Este archivo define las reglas y el flujo de trabajo para el proyecto.

## Estándares de Ingeniería
- **Pruebas:** Cobertura de pruebas unitarias superior al 80%.
- **Commits:** Confirmaciones por cada tarea completada.
- **Resúmenes:** Incluir un resumen detallado de la tarea en el cuerpo del mensaje de commit.

## Flujo de Tareas (TDD)
1. Escribir pruebas unitarias para la funcionalidad.
2. Implementar la funcionalidad para que pasen las pruebas.
3. Refactorizar y verificar cobertura.
